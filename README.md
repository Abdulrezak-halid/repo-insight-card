# Repo Insight Card

Generate profile and repository insight cards using the GitHub REST API and GitHub Actions.

Repo Insight Card is a small, production-ready GitHub integration. It fetches public GitHub user and repository data, computes useful analytics, renders an SVG profile card, writes a JSON report, and can update this README automatically from a scheduled GitHub Actions workflow.

## Features

- Uses the GitHub REST API to fetch profile and repository data.
- Generates `dist/profile-card.svg` for visible repository/profile analytics.
- Generates `dist/insights.json` for machine-readable output.
- Updates a marked README section automatically.
- Runs on push, on a daily schedule, or manually through GitHub Actions.
- Supports `GITHUB_TOKEN` authentication for higher API rate limits.
- Handles GitHub API failures and rate-limit exhaustion with typed errors.
- Emits structured JSON logs for CI and debugging.
- Includes tests, issue templates, support docs, and security guidance.

## Installation

Requirements:

- Node.js 20 or newer.
- A public GitHub account.
- Optional: a GitHub personal access token for higher API rate limits.

Clone and install:

```bash
git clone https://github.com/<your-github-username>/repo-insight-card.git
cd repo-insight-card
npm test
```

This project has no runtime npm dependencies.

## Configuration

Copy the example environment file:

```bash
cp .env.example .env
```

Environment variables:

| Variable          | Required            | Description                                                                         |
| ----------------- | ------------------- | ----------------------------------------------------------------------------------- |
| `GITHUB_USERNAME` | Yes outside Actions | GitHub username to analyze. In Actions, this defaults to `GITHUB_REPOSITORY_OWNER`. |
| `GITHUB_TOKEN`    | No                  | Token used for authenticated GitHub REST API requests and higher rate limits.       |
| `OUTPUT_DIR`      | No                  | Output directory. Defaults to `dist`.                                               |
| `README_PATH`     | No                  | README file to update. Defaults to `README.md`.                                     |
| `LOG_LEVEL`       | No                  | `debug`, `info`, `warn`, or `error`. Defaults to `info`.                            |
| `UPDATE_README`   | No                  | Set to `false` to skip README updates.                                              |

## Usage


Run tests:

## GitHub API Usage

Repo Insight Card explicitly integrates with GitHub APIs.

REST API endpoints:

| Endpoint                      | Purpose                                                                                                |
| ----------------------------- | ------------------------------------------------------------------------------------------------------ |
| `GET /users/{username}`       | Fetches public profile data, including login, display name, followers, and repository counts.          |
| `GET /users/{username}/repos` | Fetches owned repositories for stars, forks, issues, languages, topics, and recent activity analytics. |

The API client is implemented in `src/github.js`. It sends the recommended GitHub REST API headers, supports bearer token authentication, detects rate-limit exhaustion from response headers, and raises typed errors for failed requests.

## GitHub Actions

This repository includes two workflows:

- `.github/workflows/ci.yml`: runs syntax validation and tests.
- `.github/workflows/generate-insights.yml`: runs tests, calls the GitHub REST API, generates artifacts, and commits changed output files.

The insight workflow runs on:

- Pushes to `main`.
- A daily schedule.
- Manual dispatch from the Actions tab.

## API Output

`dist/insights.json` contains:

- Generation timestamp.
- Profile metadata.
- Repository totals.
- Top languages.
- Top topics.
- Recently updated repositories.

`dist/profile-card.svg` contains a visual summary suitable for a README or profile page.
